    Demo Fieldwork Apps

    Collecting data is a very important thing to do. When collecting data, it is even more important to structurize it and keep the way it is collected standardized. One way to do this is by using the Collector App provided by ESRI. For a demonstration for the Environment Department in Oslo, we set up a Collector App Map and then made Story Maps from the data that was collected. It showed the Environment Department how important it is to standardize data and what the possibilities of the data are.

    Building custom StoryMaps

    Story Maps are a great way to showcase data and present projects in an interactive way. Story Maps are customizable, meaning you can change the color and some of the fonts. But when you work at a big company like Sweco Norge AS, you might want to do more than that. Story Maps are used more often every day and by creating a template completely customized using the Sweco Visual Guidelines, the Story Maps match the company. Users recognize who the Story Map is from and will appreciete the effort put in! By downloading the source code of the Story Map, changes can be made easily. A powerful tool that is often overlooked... The Story Map about the E16 Project is almost completely customized! Check it out...

    Sweco Norge AS ArcGIS Online Thumbnail Builder

    Not only data needs to be structurized, applications need to be too! ArcGIS Online provides a number of functionalities, each serving a different purpose. To prevent getting overwhelmed by all this information, we made a thumdnail builder. This includes the name of the application or map, as well as the kind of applicaton. It creates a comprehensive library of maps and apps and it looks nice too!

    FME: Html report

    The rise of Big Data is ever going on, which is not necessarily a bad thing. The only downside of this much data is that it gets crowded and hard to understand. That's where user interface design comes in. By following a set of mantras (not rules, they are free to be interpreted any way the designer likes) a user-friendly interface can be designed. We did this for a project concerning a dahsboard. By stepping into the users shoes and listening to their wants and needs, a user-friendly interface is born! To the left is a workflow we made using wireframes. These are the bare bones of the dashboard, the markup is yet to be made.

    FME: Tunnel Drilling Model

    Not only do we know how to make data look nice, we also know our way around analyzing and manipulating it! This use case asked for a specific purpose: showcase the location of a drilling machine and the sound buffer around it. Then look for the properties inside the buffer and send them a text automatically. This way, property owners stay informed! Made entirely with FME, the model is still under construction but the value of it is big! The idea behind the model can be used for many use cases.
